🎯 项目目标
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

创建一个系统的消融实验框架，用于研究 CycleGAN 中各个损失函数组件的贡献。

特别关注点: 循环一致性损失、恒等映射损失、GAN 损失的相互作用

   包含 5 个预设消融实验：
   1. Full CycleGAN (Baseline) - 所有损失都启用
   2. No Cycle Loss - 移除循环一致性损失
   3. No Identity Loss - 移除恒等映射损失
   4. Cycle Loss Only - 仅保留循环损失
   5. Half/Double Cycle Weight - 权重变体

📊 预设实验方案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【方案 A: 基础消融 (推荐入门)】
- 运行时间: 2-3 天 (1 GPU) / 6-12 小时 (8 GPU)
- 实验数: 4 (Full + 3 消融)
- 关键问题: 各个损失函数的重要性是什么？

实验配置:
✓ Baseline: λ_A=10, λ_B=10, λ_identity=0.5
✓ No Cycle: λ_A=0, λ_B=0, λ_identity=0.5
✓ No Identity: λ_A=10, λ_B=10, λ_identity=0
✓ Cycle Only: λ_A=10, λ_B=10, λ_identity=0 (仅循环损失)

预期发现:
• 循环损失是必要的 (移除时严重失真)
• 恒等损失对特定任务重要 (尤其是颜色转换)
• GAN 损失提供现实感

【方案 B: 权重敏感性 (深入研究)】
- 运行时间: 3-4 天 (1 GPU) / 12-18 小时 (8 GPU)
- 实验数: 5 (Baseline + 4 权重变体)
- 关键问题: 最优的超参数权重是什么？

实验配置:
✓ Baseline: λ_A=10, λ_B=10
✓ Light: λ_A=5, λ_B=5
✓ Normal: λ_A=10, λ_B=10
✓ Heavy: λ_A=20, λ_B=20
✓ Extreme: λ_A=30, λ_B=30

预期发现:
• 权重 λ=10 通常最优
• 权重过低 (λ=5): 转换过度
• 权重过高 (λ=20+): 结构保守

【方案 C: 快速验证 (原型设计)】
- 运行时间: 1-2 小时 (1 GPU)
- 实验数: 2 (Baseline + 1 消融)
- 用途: 快速验证环境和流程

配置:
✓ Full (50 轮)
✓ No Cycle (50 轮)

💡 建议: 从方案 C 开始 (验证环境) → 方案 A (基础理解) → 方案 B (深入研究)
